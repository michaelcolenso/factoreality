# Product Orchestration Runbook

## Purpose
This runbook turns the medical-bill kit concept into an execution system that can be delegated across specialized agents while preserving quality, compliance, and buyer outcomes.

## Commander's Intent
Deliver the most practical self-advocacy product in this niche: buyers should be able to open the kit and take concrete action in under 30 minutes.

## Non-Negotiable Product Principles
1. **Action > explanation**: every module must include exact next steps.
2. **Scripts over theory**: include high-friction call and letter language.
3. **Defensive realism**: no guaranteed-outcome claims.
4. **Momentum design**: each asset should shorten time-to-first-win.
5. **Documentation discipline**: all workflows include logs and proof records.

## Subagent Topology
- **Orchestrator**: sequencing, dependency management, gate progression.
- **Research Analyst**: sources, statutes, policy caveats, benchmark claims.
- **Curriculum Architect**: module flow, cognitive load, progression design.
- **Template Engineer**: scripts, letters, checklists, trackers, worksheets.
- **UX Writer**: plain-language readability and confidence-boosting phrasing.
- **QA Reviewer**: rubric scoring and revision directives.
- **Compliance Editor**: legal-risk language and disclaimer checks.

## Stage Plan with Acceptance Criteria

### Stage 0 — Planning
**Inputs**: `spec.md`
**Outputs**: `plan.md`
**Acceptance criteria**:
- All deliverables mapped to owners.
- Word/page and formatting constraints reflected.
- Risks and dependencies logged.

### Stage 1 — Research and Risk Map
**Outputs**:
- Source-backed claims sheet.
- Rights/appeals/collections risk map.
- Objection inventory from audience research.
**Acceptance criteria**:
- Every statistical claim has source + date.
- Legal references include caveat language.
- No claim implies guaranteed debt reduction.

### Stage 2 — Product Architecture Lock
**Outputs**:
- Final module outline.
- Asset-to-module dependency table.
- “Quick win” path for first-time user.
**Acceptance criteria**:
- Module sequence follows urgency-first logic.
- Every module contains at least one actionable checklist.

### Stage 3 — Asset Production
**Outputs**:
- Core guide draft.
- Script pack draft.
- Letter/template library draft.
**Acceptance criteria**:
- All scripts use plain English and confidence framing.
- All templates include placeholders and filing instructions.
- No placeholders like `TBD`, `TODO`, or `INSERT` remain.

### Stage 4 — Editorial and Buyer-Outcome QA
**Outputs**:
- Edited guide.
- Buyer journey friction audit.
- “First 48 hours” implementation checklist.
**Acceptance criteria**:
- Readability at grade-8 target.
- 30-minute time-to-first-action path is explicit.
- Objection handlers present for fear, overwhelm, and low cash flow.

### Stage 5 — Formatting and Packaging
**Outputs**:
- PDF guide, template files, scripts pack, manifest.
- Tiered package bundles: Standard / Pro / Complete Defense.
**Acceptance criteria**:
- File naming and folder structure are clean and predictable.
- Every promised deliverable exists and opens correctly.
- Bundle contents match tier matrix exactly.

### Stage 6 — Launch Readiness
**Outputs**:
- Sales page copy skeleton.
- Email nurture sequence.
- 30-day content and campaign calendar.
**Acceptance criteria**:
- Messaging aligns with product claims and disclaimers.
- Analytics events defined for opt-in, checkout start, purchase, upsell.
- Initial optimization backlog created (top 10 experiments).

## Revision Protocol
If any stage fails QA:
1. Convert feedback into concrete task list.
2. Assign each task to a single owner.
3. Revise only affected files.
4. Re-run gate and document deltas.

## Buyer Delight Layer (Beyond Baseline)
To exceed expectations, add:
- **Case-path selector**: “Which scenario are you in?” navigator.
- **Call confidence cards**: one-page scripts for live calls.
- **Document tracker**: date/time/contact/proof matrix.
- **Escalation ladder**: exact trigger points for moving to higher authority.
- **72-hour rescue plan**: emergency sequence for imminent collections risk.
